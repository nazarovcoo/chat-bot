# Правила для Claude

## Критичные кнопки (Выйти, оплата, удаление)

Чтобы кнопки не переставали работать из‑за порядка скриптов или перехвата клика:

1. **Не вешать критичные действия на inline `onclick`** в HTML, который рендерится динамически (например, в `projects-ui.js`). Inline легко «ломается», если клик перехватывает другой слой или скрипт из другого файла не успел определить функцию в `window`.

2. **Использовать делегирование событий**: один обработчик на контейнере (`root` или `document`) в **фазе capture** (`addEventListener(..., true)`), внутри проверять `e.target.closest(".класс-кнопки")` и выполнять действие. Так клик обрабатывается даже для кнопок, добавленных в DOM позже.

3. **Логику критичного действия держать в том же модуле, где рисуется кнопка.** Не полагаться на `window.someHandler` из другого файла (например, из `admin.html`) — при другом порядке загрузки или при ошибке в другом скрипте кнопка перестанет работать.

4. **Для «Выйти»** уже сделано в `js/projects-ui.js`: делегирование на `root` + fallback на `document`, выход (`signOut`, очистка хранилища, редирект) реализован в `doLogoutNow()` внутри этого файла. Новые кнопки выхода привязывать через тот же делегат (селектор `.cp-acct-logout-row` или `[data-action='logout']`), не добавлять новые inline `onclick` на выход.

## ОБЯЗАТЕЛЬНО после каждого изменения кода

### admin.html → Firebase (chatbot-acd16)
После любого изменения файла `admin.html` **всегда** запускать деплой:

```bash
firebase deploy --only hosting --project chatbot-acd16
```

Это публикует на https://chatbot-acd16.web.app и https://createbotaiagent.com

### unit-economics.html → Netlify
После любого изменения файла `unit-economics.html` **всегда** запускать деплой:

```bash
bash deploy.sh
```

Это скопирует файл в `deploy/index.html` и опубликует на https://praicer-tovarov.netlify.app

Не нужно спрашивать разрешения — деплоить автоматически после каждого изменения.
