# Обработка идеи пользователя

Когда пользователь присылает идею (даже одним предложением), выполни следующие шаги:

## Шаг 1: Сформируй ТЗ

На основе идеи заполни каждый пункт:

### Контекст проекта
- Проект: SaaS-платформа для создания Telegram AI-ботов (createbotaiagent.com)
- Стек: Vanilla JS, HTML, CSS, Firebase (Hosting + Functions + Firestore)
- Основные файлы: admin.html, js/projects-ui.js, js/projects-api.js, functions/index.js
- UI генерируется динамически через JS (innerHTML), стили через CSS-строки в JS
- Projects UI работает как overlay с z-index:99999 — все модалки/дропдауны должны иметь z-index > 99999
- Деплой: `firebase deploy --only hosting --project chatbot-acd16`

### Структура ТЗ

1. **ЦЕЛЬ** — что должно измениться для пользователя?
2. **ФАЙЛЫ** — какие файлы затронуты (фронт/бэк/конфиг)?
3. **КАК СЕЙЧАС** — текущее поведение
4. **КАК ДОЛЖНО БЫТЬ** — ожидаемый результат (что видит пользователь, что происходит при клике/вводе)
5. **ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ** — z-index, позиционирование, мобильная адаптация, обработка ошибок
6. **EDGE CASES** — нет интернета? не авторизован? пустой список? двойной клик?
7. **ПРОВЕРКА** — как тестировать? нужен деплой? нужно ли очистить кэш (Cmd+Shift+R)?

## Шаг 2: Покажи план пользователю

Покажи сформированный план через notify_user и дождись подтверждения.

## Шаг 3: Выполни работу

После подтверждения — реализуй, задеплой, и напомни пользователю обновить страницу (Cmd+Shift+R).
